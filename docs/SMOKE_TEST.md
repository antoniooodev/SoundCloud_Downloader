# Smoke Test Guide

## Purpose

This guide walks through a local MVP smoke test for one authorized SoundCloud
track. It is intended to verify configuration, OAuth setup, token storage, and
one policy-approved local artifact write.

Use only tracks your account is authorized to access. DRM-protected or
encrypted streams are denied fail-closed, and this project does not support
protected stream decryption or DRM bypass.

## Prerequisites

- Python 3.11 or newer.
- `ffmpeg` available on `PATH`.
- SoundCloud OAuth app credentials for your own account.
- A SoundCloud track URL that your account is authorized to access.

Do not paste real tokens, authorization codes, client secrets, Fernet keys, or
private SoundCloud URLs into documentation, issues, or commits.

## 1. Create a virtual environment

```bash
python -m venv .venv
```

## 2. Install development dependencies

```bash
PYTHON=.venv/bin/python make install-dev
```

## 3. Create local configuration

```bash
cp .env.example .env
```

Edit `.env` locally. Replace placeholder values with your own local OAuth app
settings and keep both safety gates enabled only for the smoke test:

```dotenv
SCD_ALLOW_NETWORK=true
SCD_ALLOW_FILESYSTEM_WRITES=true
SCD_SOUNDCLOUD_CLIENT_ID=<your-soundcloud-client-id>
SCD_SOUNDCLOUD_CLIENT_SECRET=<your-soundcloud-client-secret-if-required>
```

Output artifacts are written under `SCD_ARTIFACT_STORAGE_ROOT`.

## 4. Generate encryption keys

Generate one Fernet key for the OAuth session store and a separate key for the
OAuth token store:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Put the generated local values into `.env`:

```dotenv
SCD_OAUTH_SESSION_ENCRYPTION_KEY=<generated-local-fernet-key>
SCD_OAUTH_TOKEN_ENCRYPTION_KEY=<generated-local-fernet-key>
```

Do not commit the generated keys.

## 5. Check configuration

```bash
soundcloud-downloader doctor --env-file .env
```

Resolve reported errors before continuing. The doctor command may check whether
the configured `ffmpeg` binary is discoverable; it does not execute media
conversion. It also reports `download_ready` and a safe `missing_required` list
for track download prerequisites.

## 6. Create OAuth session

```bash
soundcloud-downloader oauth create-session \
  --env-file .env \
  --client-id "<your-soundcloud-client-id>" \
  --redirect-uri "https://your-app.example/callback" \
  --persist \
  --allow-filesystem-writes
```

Open the printed authorization URL in your browser and complete authorization
with your own SoundCloud account.

## 7. Exchange authorization code

After your redirect receives an authorization code and state value, exchange
them locally:

```bash
soundcloud-downloader oauth exchange-code \
  --env-file .env \
  --session-id "<session-id-from-create-session>" \
  --code "<code-from-callback>" \
  --state "<state-from-callback>" \
  --allow-network \
  --allow-filesystem-writes
```

Use only the short-lived code from your own local authorization flow. Do not
store or commit authorization codes.

## 8. Check token status

```bash
soundcloud-downloader oauth token-status --env-file .env
```

The output should report whether the configured token profile is present and
expired without printing token values.

## 9. Download one authorized track

```bash
soundcloud-downloader download track "https://soundcloud.com/example/track" \
  --env-file .env \
  --format m4a \
  --profile-id default \
  --allow-network \
  --allow-filesystem-writes
```

Replace the example URL with one track your account is authorized to access.
Protected or encrypted sources are denied safely instead of being decrypted.

## 10. Verify output artifact

Inspect the command output for the artifact path and checksum. The artifact
should be under `SCD_ARTIFACT_STORAGE_ROOT`.

```bash
soundcloud-downloader doctor --env-file .env
```

Confirm `download_ready` is `true` or review `missing_required` for any
remaining symbolic prerequisites.

## 11. Logout

Remove the local encrypted token entry when you are done:

```bash
soundcloud-downloader oauth logout \
  --env-file .env \
  --allow-filesystem-writes
```

## Troubleshooting

- `allow_network is false` means `SCD_ALLOW_NETWORK=true` is missing or was not
  passed through the CLI safety flag for that command.
- `allow_filesystem_writes is false` means `SCD_ALLOW_FILESYSTEM_WRITES=true`
  is missing or was not passed through the CLI safety flag for that command.
- Missing client credential errors mean the local `.env` still contains
  placeholders or omitted required SoundCloud app settings.
- Fernet key errors mean the encryption key placeholders were not replaced with
  values generated by `cryptography.fernet.Fernet.generate_key()`.
- DRM, encrypted, preview-only, or rights-restricted sources are expected to
  fail closed. Use another track that your account is authorized to access.
